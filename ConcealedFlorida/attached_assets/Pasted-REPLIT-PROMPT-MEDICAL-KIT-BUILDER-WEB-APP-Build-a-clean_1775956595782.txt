REPLIT PROMPT — MEDICAL KIT BUILDER WEB APP
Build a clean, responsive web application called "Build Your Med Kit" with the following features and structure:

OVERVIEW
Three tiers of medical kits:

Basic – Everyday booboos (cuts, scratches, minor burns)
Medium – Trauma-capable car/bag kit
Advanced – Full trauma kit for serious emergencies

Each tier has:

A visual checklist page (printable, with checkboxes, laminate-friendly)
Bag recommendations with dimensions + Amazon/NAR links (with fallback dimensions if links die)
Item cards with name, quantity, expiration info, and purchase links
A pill/medication ziplock section with a printable name plate showing: product name, expiration date, contents, and owner name field
A "Buy on Amazon" / "Buy on NAR" button per item

PRINTABLE CHECKLIST FEATURE
Each tier must have a "Print Checklist" button that opens a print-friendly page with:

The kit name and tier at the top
All items listed with ☐ checkbox next to each
Expiration date field next to expirable items: Exp: ________
A footer with: Last checked: _________ | Next check due: _________
Clean black-and-white layout suitable for laminating (no colored backgrounds)
Font size minimum 12pt for readability when laminated
Section dividers: OUTSIDE BAG / INSIDE BAG / PILLS / EXTRAS / BANDAGES


GENERAL UI REQUIREMENTS

Mobile-friendly responsive layout
Each tier has its own tab or page
Items display as cards with: item name, quantity, expiration info, and 1–2 buy links
Color coding: Green = no expiration | Yellow = expires 3–5 years | Orange = expires 1–2 years | Red = expires under 1 year
"Mark as Owned" toggle per item (saves to localStorage)
Progress bar per kit showing % of items owned
Bag dimension specs always visible even if buy link is broken
Footer links to: Stop The Bleed training, Red Cross CPR training, PrepMedic YouTube


TECH STACK SUGGESTION

React or plain HTML/CSS/JS
No backend needed (localStorage for state)
Print styles via @media print CSS
Keep it single-page or multi-tab
